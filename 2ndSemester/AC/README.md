# AC — Computer Architecture

## Overview

Computer Architecture (AC) explores how computers are structured and how programs are executed at the hardware level.  
The course focuses on understanding the relationship between software and hardware, from instruction execution to memory organization.

It deepens the knowledge introduced in Digital Systems and connects low-level logic to real processor design.

## Topics Covered

- Data representation (binary, two’s complement, floating point)  
- Instruction set architecture (ISA)  
- CPU organization and datapath  
- Control unit design  
- Assembly language basics  
- Memory hierarchy (cache, RAM)  
- Input/Output mechanisms  

## Learning Objectives

By the end of the course, the student should be able to:

- Understand how instructions are executed internally  
- Interpret simple assembly programs  
- Analyze CPU components and datapath design  
- Explain memory hierarchy and performance trade-offs  

## Relevance to the Degree

AC is fundamental for understanding how software runs on real machines.  
It is especially important for:

- Systems programming  
- Embedded systems  
- Operating systems  
- Performance optimization  

A solid grasp of low-level execution models is essential for advanced system-level development.
