# TDS — Software Development Techniques

## Overview

Software Development Techniques (TDS) focuses on structured software design and engineering practices.  
The course emphasizes writing maintainable, testable, and scalable code using sound architectural principles.

It marks the transition from simply implementing features to designing robust software systems.

## Topics Covered

- Software design principles (SOLID, separation of concerns)  
- Layered architecture  
- Design patterns  
- Unit testing and test-driven development  
- Refactoring techniques  
- Dependency management  
- Code quality and maintainability  

## Learning Objectives

By the end of the course, the student should be able to:

- Design modular and extensible software systems  
- Apply common design patterns appropriately  
- Write unit tests and structure code for testability  
- Refactor code to improve clarity and maintainability  

## Relevance to the Degree

TDS is fundamental for professional software development.  
It is especially important for:

- Backend development  
- Large-scale applications  
- Software architecture  
- Team-based development  

The course reinforces disciplined engineering practices and prepares students for real-world software projects.
